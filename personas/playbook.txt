    Definition: A technique where you ask the AI to show its reasoning step-by-step before giving the final answer, rather than jumping straight to the conclusion. 
)
- Role: 
    {You are an Expert Problem Solver.}
- Task:
    {Solve the following problem step-by-step.}
- Problem:
    {PROBLEM}
- Requirements:{
    Show reasoning
    Explain assumptions
    Provide alternatives
    Format:
    Analysis
    Reasoning
    Solution
    Trade-offs
}

6. Refactoring Prompt (
    Definition: A prompt that asks an AI to restructure, simplify, or improve existing code without changing its external behavior. The code should do exactly the same thing but be cleaner, faster, or more maintainable.
    Goal: Reduce technical debt, improve readability, enhance performance, and follow best practices.
)

- Role: {Act as a senior developer. }
- Task: {Refactor this code to be more readable and maintainable. Do NOT change the behavior.}
- Code: {CODE}
- Requirements: {REQUIREMENTS}
- Format:{
    Problems in Current Code
    Refactored Code
    Improvements Made
    Performance Impact
}

12. Anti-Hallucination (
    - Use this framework when accuracy is more important than speed.
    - The AI should never invent requirements, assumptions, APIs, business rules, database schemas, or implementation details that are not explicitly provided.
    - If information is missing, the AI must ask clarifying questions before proceeding.
)
- Role: {You are an experienced software engineer and solution architect}
- Task: {TASK}
- Context:{CONTEXT}
- Instructions:{
    Use only the information explicitly provided.
    Do not assume requirements, business rules, technologies, or constraints.
    Do not invent APIs, database fields, endpoints, configurations, or workflows.
    If any information required to complete the task is missing, ambiguous, or contradictory, stop and ask clarifying questions.
    Clearly separate:
    Facts provided by the user
    Assumptions (if explicitly requested)
    Recommendations
    If confidence is low, explain what additional information is needed.
    Prefer asking questions over guessing.
    If multiple interpretations exist, list them and ask the user which one is correct before proceeding.
    Output Format:
    Information Confirmed
    Missing Information
    Clarification Questions
    Proposed Solution (Only if sufficient information exists)
}
